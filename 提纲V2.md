# 财务模型知识图谱 — 面向财务咨询公司演示提纲 V2

## Context

领导反馈：不要一上来讲痛点，先讲大图。核心故事是 **Excel → 图谱 → 修改 → 导回 Excel**。痛点是"转成图谱后解决的问题"，不是开场白。

---

## 一、大图：我们做了什么（3 分钟）

> 目标：一句话讲清"这是什么"

**核心故事**：Excel 财务模型 ↔ 知识图谱 ↔ Excel 财务模型

```
[Excel 财务模型]  ──解析──▶  [知识图谱]  ──导出──▶  [Excel 财务模型]
                                    │
                              修改参数 / 重算 / 问答
```

三句话讲完：
1. **输入**：一个 .xlsx 文件丢进来
2. **中间**：转成知识图谱，看见所有公式依赖、指标关系
3. **输出**：改完后，导回 .xlsx，还是一个能用的财务模型

**关键信息**：这不是替代 Excel，而是给 Excel 加一个"可理解、可操作、可追溯"的中间层。

**过渡话术**：「为什么要做这件事？因为 Excel 模型有几个天然的问题。」

---

## 二、为什么要做：Excel 模型的四个天然问题（5 分钟）

> 目标：痛点放在"大图之后"，作为"为什么要建图谱"的理由

| # | 问题 | 图谱怎么解 |
|---|------|------------|
| 1 | **黑盒**：打开 50+ Sheet 不知道数字怎么来的 | 图谱可视化依赖链，一键追溯上下游 |
| 2 | **影响不可控**：改一个参数，不知道影响多少格 | 改前自动识别影响范围，改后对比差异 |
| 3 | **知识沉淀难**：模型做完死在文件里 | 图谱是可复用的结构化知识资产 |
| 4 | **版本对比难**：两个版本只能肉眼找不同 | 结构化 diff + 影响链可视化 |

**要点**：每个问题都对应"图谱能做什么"，不只是抱怨 Excel。

**过渡话术**：「基于这些需求，我们开发了四个能力。下面逐一演示。」

---

## 三、四个能力演示（12 分钟）

### 能力 1：解析 — 一键转图谱（3 分钟）
- 操作：上传 .xlsx → 自动解析 → 展示图谱统计
- 数据：49 张表、2968 指标、35000+ 单元格、73% 公式覆盖
- 价值：10 分钟人工工作 → 10 秒自动完成

### 能力 2：浏览 — 交互式依赖追踪（3 分钟）
- 操作：打开图谱浏览器 → 点击节点 → 展开上下游
- 看点：Sheet → Table → Indicator → Cell 逐层下钻
- 价值：回答"这个数字怎么来的"

### 能力 3：修改 — 改参数看影响（3 分钟）
- 操作：选择场景 → 修改参数 → 重算 → 查看影响
- 看点：自动识别受影响单元格，修改前后对比
- 价值：改一个数，立刻知道影响范围

### 能力 4：导出 — 改完导回 Excel（3 分钟）
- 操作：修改后的图谱 → 导出 .xlsx → 打开验证
- 看点：导出的文件保留原公式结构和格式
- 价值：闭环 — 从 Excel 来，回 Excel 去

> **注意**：能力 4 是领导强调的"最早需求"，单独作为一个能力展示，不再隐藏。

---

## 四、进阶能力：快照对比 + LLM 问答（5 分钟）

### 快照对比（2 分钟）
- 选两个快照 → 热力图展示差异 → 传播路径可视化
- 场景：方案 A vs 方案 B 对比

### LLM 智能问答（3 分钟）
- 提问 → 结构化回答（指标卡 + 折线图 + 数据表）
- 示例："2026年营业收入多少？"
- 价值：客户提问不再翻表找数

---

## 五、技术可靠性（2 分钟）

> 一页讲清技术能力，建立信任

- 公式解析：73% 覆盖，依赖方向 100% 正确
- 重算精度：#VALUE! 从 5557 降至 0
- 浮点精度：相对+绝对容差，避免 0 vs -3.64e-11 误判
- 数据隔离：多任务独立，本地部署，数据不外传

- 能否处理合并单元格？（是）

- 能否接入已有模型？（任何 .xlsx 均可）
