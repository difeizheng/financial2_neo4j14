## 二、优化方案（按收益排序）

### 2.1 ⭐ 预编译 Input Map（预期收益：50-70%，最大杀手）

**问题根因：**

`_build_input_map` 每次 `evaluate_cell` 调用都重新做：
1. 解析 sheet 名 → `_normalize_sheet_name()` → 遍历 graph 的所有 sheet 名匹配
2. 拆地址 → `_split_col_row()` → 正则匹配
3. 展开范围 → `_resolve_range()` → 循环所有 cell_id → 创建 numpy 数组

这些操作的结果**完全由公式决定，与值无关**。同一个公式 `=SUM(F5:BA5)`，不管调 1 次还是 100 次，引用的永远是那 49 个 cell_id。

**优化方案：将 input 解析分为两层**

```
结构层（static，只做一次）：
  公式 → input_keys → [cell_id_1, cell_id_2, ..., cell_id_N]
  缓存 key = (cell_id, formula_raw)

值层（dynamic，每次评估）：
  [cell_id_1, ..., cell_id_N] → 读取 graph.cells 中的值 → numpy 数组
```

**实现代码：**

```python
# evaluator.py

# 缓存：(cell_id) → InputPlan
# InputPlan 记录了"这个公式引用哪些 cell，每个是单值还是范围"
from dataclasses import dataclass, field

@dataclass
class _RangePlan:
    cell_ids: list[str]        # 展开后的 cell_id 列表
    shape: tuple[int, int]     # numpy 数组形状 (rows, cols)

@dataclass
class _InputPlan:
    """结构信息：一个公式的所有输入参数如何映射到 cell_id"""
    scalar_inputs: dict[str, str]       # input_key → cell_id
    range_inputs: dict[str, _RangePlan] # input_key → 范围信息
    sheet: str                          # 所属 sheet

_input_plans: dict[str, _InputPlan] = {}

def _get_or_build_plan(cell_id: str, func, formula_sheet: str, graph: FinancialGraph) -> _InputPlan:
    if cell_id in _input_plans:
        return _input_plans[cell_id]

    plan = _InputPlan(scalar_inputs={}, range_inputs={}, sheet=formula_sheet)

    for raw_key in func.inputs:
        if "!" in raw_key:
            sheet_part, addr_part = raw_key.split("!", 1)
            sheet_part = sheet_part.strip("'")
            sheet_part = _normalize_sheet_name(sheet_part, graph)
        else:
            sheet_part = formula_sheet
            addr_part = raw_key

        addr_part = addr_part.replace("$", "")

        if ":" in addr_part:
            # 范围：预计算 cell_id 列表
            start_col, start_row, end_col, end_row = _parse_range_addr(addr_part)
            from openpyxl.utils import column_index_from_string, get_column_letter
            sc = column_index_from_string(start_col)
            ec = column_index_from_string(end_col)
            sr, er = int(start_row), int(end_row)

            cell_ids = []
            for r in range(sr, er + 1):
                for c in range(sc, ec + 1):
                    col_letter = get_column_letter(c)
                    cell_ids.append(f"{sheet_part}_{r}_{col_letter}")

            plan.range_inputs[raw_key] = _RangePlan(
                cell_ids=cell_ids,
                shape=(er - sr + 1, ec - sc + 1),
            )
        else:
            # 标量：直接记录 cell_id
            cid = _addr_to_cell_id(sheet_part, addr_part)
            plan.scalar_inputs[raw_key] = cid

    _input_plans[cell_id] = plan
    return plan


def _build_input_map_fast(plan: _InputPlan, graph: FinancialGraph) -> dict[str, np.ndarray]:
    """从 plan 快速构建 input map：只读值，不做解析"""
    kwargs: dict[str, np.ndarray] = {}

    for key, cell_id in plan.scalar_inputs.items():
        cell = graph.cells.get(cell_id)
        kwargs[key] = np.array([[_coerce(cell.value if cell else None)]])

    for key, rp in plan.range_inputs.items():
        rows_data = []
        idx = 0
        for r in range(rp.shape[0]):
            row_vals = []
            for c in range(rp.shape[1]):
                cell = graph.cells.get(rp.cell_ids[idx])
                row_vals.append(_coerce(cell.value if cell else None))
                idx += 1
            rows_data.append(row_vals)
        kwargs[key] = np.array(rows_data)

    return kwargs


# 修改 evaluate_cell：
def evaluate_cell(cell_id: str, graph: FinancialGraph):
    cell = graph.cells.get(cell_id)
    if cell is None or not cell.formula_raw:
        return None

    func = _compile_formula(cell.formula_raw)
    if func is None:
        return None

    plan = _get_or_build_plan(cell_id, func, cell.sheet, graph)
    kwargs = _build_input_map_fast(plan, graph)
    result = func(**kwargs)
    return _extract_scalar(result)
```

**收益分析：**

假设 20K cells，SCC 内每个 cell 平均迭代 5 次：
- 原方案：20K × 5 × (_build_input_map 开销) = 100K 次完整解析
- 新方案：20K × 1 × (结构解析) + 100K × (只读值) = 20K 次解析 + 100K 次快速读值

结构解析占 `_build_input_map` 的 ~70% 时间，所以总节省 ≈ 100K × 70% / 100K = **~50-70%**

**缓存失效时机：** `clear_formula_cache()` 时同时清 `_input_plans`

### 2.2 ⭐ SCC 迭代中直接遍历 dirty 集合（预期收益：减少无效循环）

**当前代码问题：**

```python
# recalculator.py _converge_sccs 中
for cell_id in sorted_cyclic:       # ← 遍历全部环内格
    if cell_id not in dirty:         # ← 跳过非脏格
        continue
```

如果环内有 1000 个格，但只有 10 个是 dirty 的，每次迭代仍然遍历 1000 个格做集合检查。

**修复：直接遍历 dirty 集合**

```python
for cell_id in list(dirty):         # ← 只遍历脏格
    cell = graph.cells.get(cell_id)
    ...
```

这是一个简单的改动，但对大环（如财务模型中的资金筹措循环）效果显著。

### 2.3 ⭐ 简单公式快速路径（预期收益：对简单公式提速 10-50x）

**问题：** `formulas` 库是通用 Excel 引擎，对简单公式有大量开销（AST 遍历、numpy 包装、异常处理）。

大量单元格的公式其实很简单：
- `=A1+B1`（算术）
- `=SUM(F5:BA5)`（求和）
- `=IF(A1>0, B1, 0)`（条件）
- `=-A1`（取反）

**方案：对简单模式做 regex 快速路径**

```python
import operator

_SIMPLE_BINOPS = {
    '+': operator.add, '-': operator.sub,
    '*': operator.mul, '/': operator.truediv,
}

def _try_fast_eval(formula: str, plan: _InputPlan, graph: FinancialGraph):
    """对简单公式直接计算，跳过 formulas 库。"""

    # 模式 1：单值取反  =-REF
    m = re.match(r'^=-(\w+!?\$?[A-Z]+\$?\d+)$', formula)
    if m:
        val = _read_scalar(m.group(1), plan, graph)
        return -val if val is not None else None

    # 模式 2：二元运算  =REF op REF
    m = re.match(r'^=(\w+!?\$?[A-Z]+\$?\d+)([+\-*/])(\w+!?\$?[A-Z]+\$?\d+)$', formula)
    if m:
        a = _read_scalar(m.group(1), plan, graph)
        b = _read_scalar(m.group(3), plan, graph)
        if a is not None and b is not None:
            return _SIMPLE_BINOPS[m.group(2)](a, b)

    # 模式 3：SUM(range)
    m = re.match(r'^=SUM\(([A-Z]+\d+:[A-Z]+\d+)\)$', formula, re.IGNORECASE)
    if m:
        vals = _read_range_values(m.group(1), plan, graph)
        return sum(v for v in vals if isinstance(v, (int, float)))

    # 模式 4：标量运算 =REF*0.5
    m = re.match(r'^=(\w+!?\$?[A-Z]+\$?\d+)([*/])([\d.]+)$', formula)
    if m:
        val = _read_scalar(m.group(1), plan, graph)
        scalar = float(m.group(3))
        if val is not None:
            return _SIMPLE_BINOPS[m.group(2)](val, scalar)

    return _FAST_EVAL_MISS  # sentinel: 不匹配任何快速路径
```

**收益估算：** 先统计模型中公式分布：

```python
from collections import Counter
formula_patterns = Counter()
for cell in graph.cells.values():
    if cell.formula_raw:
        f = cell.formula_raw.lstrip('=')
        if re.match(r'^-[A-Z]', f):
            formula_patterns['unary_neg'] += 1
        elif re.match(r'^[A-Z]+\d+[+\-*/][A-Z]+\d+$', f):
            formula_patterns['binop'] += 1
        elif re.match(r'^SUM\(', f, re.IGNORECASE):
            formula_patterns['sum'] += 1
        else:
            formula_patterns['complex'] += 1
print(formula_patterns)
```

如果 70% 的公式能走快速路径，且快速路径比 `formulas` 快 20x，总加速 ≈ **3-5x**。

acyclic_set = set(acyclic_affected)

**注意：** 这个优化只有在环外格很多且环内变化只影响少数环外格时才有收益。如果几乎所有环外格都直接依赖环内格，收益不大。

### 2.5 拓扑排序增量优化（文档原有，确认是否需要）

**诊断方法：**

```python
import time
t0 = time.perf_counter()
affected = downstream_cells(graph, updates.keys())
print(f"downstream_cells: {time.perf_counter()-t0:.2f}s, {len(affected)} cells")
```

如果这一步 < 1s，不值得优化。如果 > 5s，考虑：

1. **预计算下游关系索引**：对每个 cell 预存 `downstream[cell_id] = set of dependent cells`，避免每次 BFS
2. **增量排序**：只对受影响的子图排序，不排序全图

### 2.6 替换 formulas 库（终极方案，收益最大）

**根本问题：** `formulas` 是纯 Python，每 cell 23ms 是解释型语言的极限。

**替代方案：**

| 方案 | 语言 | 预期加速 | 成熟度 |
|------|------|----------|--------|
| [ironcalc](https://github.com/nickelc/ironcalc) | Rust (Python binding) | 50-100x | 新项目，函数覆盖不全 |
| [CalcEngine](https://github.com/nickelc/calc) | Rust | 50-100x | 实验性 |
| 自写 C++/Rust 核心引擎 | C++/Rust | 100x+ | 开发量大 |
| 用 LibreOffice headless 计算 | C++ | 10-50x | 成熟但集成复杂 |
| 对关键函数（IRR/NPV/PMT）写 Numba | Python+LLVM | 10-100x（仅这几个函数） | 实用 |

**务实建议：** 先做 2.1-2.3 把时间降到 2-3 分钟。如果需要秒级响应，再考虑替换 formulas。
