# 抽水蓄能电站财务模型重算引擎技术文档

## 文档目的

本文档面向财务模型开发人员，解释重算引擎的设计原理、核心概念、技术实现细节，便于后续维护和功能扩展。

---

## 一、背景与问题

### 1.1 业务场景

抽水蓄能电站财务模型是一个大型 Excel 模型（~58K 单元格，49 张表，2968 个指标），包含：
- 资金筹措及还本付息表
- 利润表、现金流量表
- 敏感性分析、盈亏平衡分析
- IRR、NPV、DSCR 等核心财务指标

当用户修改某个参数（如上网电价±5%）时，需要重新计算整个模型，得出新的 IRR、NPV 等指标。

### 1.2 历史问题

原始实现存在三个导致计算结果不一致或性能极差的问题：

**问题 1：公式缓存每次清空（性能杀手）**
```
原代码：recalculate() 开头调用 clear_formula_cache()
```
- `formulas` 库编译公式比评估公式贵 10-50 倍
- 每次重算都重新编译全部公式，导致 11 小时运行时间
- 浮点编译器每次结果略有差异，引起结果不一致

**问题 2：循环依赖收敛逻辑有缺陷**
- 迭代逻辑为 `range(10)` 外层循环 + 独立的 final convergence pass
- dirty-tracking 不精确，导致重复计算和遗漏
- 与 Excel 的迭代收敛结果存在偏差

**问题 3：依赖链方向理解错误**
- 图的边方向：`add_edge(A, B)` 表示 "A 依赖 B"（A 的公式引用 B）
- 找下游受影响格时，应查 `predecessors`，不是 `successors`
- 方向反了会导致部分格没被重算，用的是旧值

## 二、核心概念

### 2.1 单元（Cell）

Excel 中的一个格子，如 `表1_250_I` 表示「表1」Sheet 第250行 第I列。

每个 Cell 包含：
- `id`：唯一标识，格式为 `{sheet}_{row}_{col}`
- `value`：当前计算值
- `formula_raw`：原始公式字符串（如 `=SUM(F10:BA10)`）
- `dependencies`：这个格子依赖哪些其他格子
- `dependents`：哪些格子依赖这个格子

### 2.2 依赖图（Dependency Graph）

用 NetworkX 有向图表示所有单元格之间的依赖关系：

```python
cell_graph.add_edge("表1_249_I", "表1_250_I")
# 表示：表1_250_I 这个格依赖 表1_249_I
# 当 表1_249_I 的值变化时，表1_250_I 需要重新计算
```

**边方向规则（重要）：**
```
add_edge(A, B)  →  "A depends on B"  →  "A 的公式引用了 B"
                 →  "B 变化时，A 需要重新算"
                 →  在图中，B 是 A 的 predecessor（入边来源）
```

### 2.3 上游与下游

给定一个被修改的格子：

```
上游（Upstream）：直接/间接影响它的格子（它的 predecessors）
下游（Downstream）：直接/间接被它影响的格子（它的 predecessors）

Edge direction: A → B means "A depends on B"
Therefore: if B changes, A must recalculate (A is downstream of B)
```

**记忆技巧：** 水往低处流，值往「被依赖的方向」传播。B 变了，所有「依赖 B 的格」都要重新算。

### 2.4 有向无环图（DAG）与环（Cycle）

**DAG（Directed Acyclic Graph）：**
- 没有环的依赖图
- 可以用拓扑排序确定计算顺序
- 每一格只算一次

**环（Cycle）：**
- 两个或多个格子互相引用，形成闭环
- 例如：A 引用 B，B 引用 C，C 引用 A
- 无法用简单拓扑排序，必须迭代收敛

```
     ┌──────┐
     ▼      │
   A ──→ B  │
   ↑      │  ← 形成环：无法确定谁先算
   │      ▼
   └─── C ←┘

典型场景：累计折旧 = 期初值 - 本年折旧，本年折旧又依赖累计折旧
```

### 2.5 强连通分量（SCC）

环内的格子集合称为「强连通分量」（Strongly Connected Component）。

```
A ↔ B ↔ C  形成一个 SCC {A, B, C}

D 引用 A，E 引用 B，D 和 E 不在环内

SCC 内部必须一起迭代收敛
SCC 之间按依赖关系排序
```

### 2.6 拓扑排序（Topological Sort）

对 DAG 进行排序，保证「被依赖的格永远在前面」。

```
拓扑排序结果示例：
  [C, B, A, D, E]  ← C 和 B 在 A 前，D 和 E 在最后
  意味着：C 和 B 的值确定后，才能算 A；A 算完后，才能算 D 和 E
```

### 2.7 dirty set（脏格追踪）

**dirty = "这个格在上轮迭代中值变了，需要重新算"**

- 初始 dirty = 环内所有格（全量）
- 每轮迭代后，根据「值是否真的变了」更新 dirty set
- 下一轮只算 dirty 集合中的格，避免无效计算

**为什么不每次算全部？**
```
假设环有 100 个格，第 1 轮只有格 #37 变了

旧逻辑：第 2 轮重新算全部 100 个格（大量浪费）
新逻辑：第 2 轮只算 #37 的后继格（精确追踪）
```

## 三、算法流程

### 3.1 总体流程图

```
┌─────────────────────────────────────────────────────────────┐
│                     recalculate(graph, updates)             │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
      ① 应用种子变更                    ② 找下游受影响格
   (修改参数格的值)                   (downstream_cells BFS)
              │                               │
              └───────────────┬───────────────┘
                              ▼
              ┌───────────────────────────────┐
              │   ③ 分离：环内格 vs 环外格    │
              └───────────────────────────────┘
                    │                 │
          ┌─────────┴───────┐     ┌───┴─────────┐
          ▼                 ▼     ▼             ▼
    ④ 环外格          ⑤ SCC迭代     (环外格在
   (无环依赖)       (收敛)         收敛后再算)
          │                 │
          └────────┬────────┘
                   ▼
              ⑥ Indicator同步
           (更新指标层)
                   │
                   ▼
              返回 RecalcResult
```

### 3.2 详细步骤

#### 步骤 ①：应用种子变更

```python
for cell_id, new_val in updates.items():
    cell = graph.cells.get(cell_id)
    if cell is None:
        continue
    old_val = cell.value
    cell.value = new_val  # 直接修改值
```

用户改了什么格，格的值就变成什么。

#### 步骤 ②：找下游受影响格

```python
affected: list[str] = downstream_cells(graph, updates.keys())
# BFS 从被修改的格出发，沿依赖链反向传播
# 找所有「依赖这些格」的格子（predecessors）
# 返回拓扑排序后的列表
```

**为什么用 BFS + 拓扑排序？**
- BFS 保证覆盖所有直接或间接依赖
- 拓扑排序保证计算顺序正确（「被依赖的先算」）

#### 步骤 ③：分离环内格和环外格

```python
for scc in nx.strongly_connected_components(subgraph):
    members_in_affected = scc & affected_set
    if len(members_in_affected) > 1:
        cyclic_cells.update(members_in_affected)  # 环内
        sccs_to_converge.append(sorted(members_in_affected))
    else:
        acyclic_affected.append(...)  # 环外
```

**判断逻辑：** 如果一个 SCC 只有一个格（在 affected 集合中），它不形成环，可以单次计算。

#### 步骤 ④：环外格单次评估

```python
for cell_id in acyclic_affected:
    cell = graph.cells.get(cell_id)
    if not cell.formula_raw:
        continue  # 常量格，跳过
    old_val = cell.value
    new_val = evaluate_cell(cell_id, graph)  # formulas 库计算
    cell.value = new_val
```

**关键：** 环外格没有循环依赖，拓扑排序保证所有前序格已算完，直接算一遍即可。

#### 步骤 ⑤：SCC 迭代收敛（核心算法）

```python
def _converge_sccs(...):
    dirty: set[str] = set(cyclic_cells)  # 初始全dirty

    for _outer in range(max_iter):  # 最多100轮
        # ── 5a. 收敛环内格 ──────────────────────────────
        max_delta = 0.0
        next_dirty: set[str] = set()

        for iter_count in range(1, max_iter + 1):
            if not dirty:
                break  # 已收敛，提前结束

            for cell_id in dirty:
                cell = graph.cells.get(cell_id)
                old_val = cell.value
                new_val = evaluate_cell(cell_id, graph)
                cell.value = new_val

                delta = abs(float(new_val) - float(old_val))
                max_delta = max(max_delta, delta)

                if delta > tol:
                    # 值变了，通知后继格下轮重新算
                    for succ in g.successors(cell_id):
                        if succ in cyclic_cells:
                            next_dirty.add(succ)

            dirty = next_dirty
            if max_delta <= tol:
                break  # 收敛

        # ── 5b. 重新算环外依赖格 ─────────────────────────
        for cell_id in acyclic_affected:
            new_val = evaluate_cell(cell_id, graph)
            if new_val is None:
                continue
            cell.value = new_val

        # ── 检查是否完成 ────────────────────────────────
        if max_delta <= tol and not nc_changed:
            break

    return total_scc_iters
```

**Unified alternating loop 解释：**

```
外层循环（_outer）：
  ├─ 内层循环1：收敛环内格
  │    每次只算dirty的格，追踪max_delta
  │    delta ≤ tol 时认为收敛
  │
  └─ 内层循环2：重新算环外依赖格
       因为环收敛后，环内格值变了
       环外格可能因此也变了
       需要重新算一遍确认

如果内外都没格再变 → 真正收敛，退出
否则 → 继续外层下一轮
```

**dirty set 工作原理：**

```
第1轮开始：
  dirty = {A, B, C}          ← 初始：环内全部格都是脏的

第1轮内：
  算A → A值变了 → dirty_next.add(B)   ← B依赖A
  算B → B值变了 → dirty_next.add(C)   ← C依赖B
  算C → C值没变 → 不通知任何人

第1轮结束：
  dirty = {B}               ← 只有B是脏的（实际变了的）

第2轮开始：
  dirty = {B}
  算B → B值变了 → dirty_next.add(A)
  算C（不在dirty中）→ 跳过

第2轮结束：
  dirty = {A}

第3轮开始：
  dirty = {A}
  算A → A值没变 → dirty_next = {}
  收敛完成
```

## 四、公式评估器

### 4.1 职责

`evaluator.py` 负责把一个 Cell 的公式字符串转换回数值。

**核心挑战：** Excel 引用格式 vs Cell ID 格式

```
Excel 公式引用：F5, $I$250, '表1-资金筹措'!I250, F5:BE5
Cell ID 格式：  表1_250_I
```

`formulas` 库使用 Excel 格式，我们的图用 Cell ID 格式，中间需要转换层。

### 4.2 公式编译缓存

```python
_compiled_cache: dict[str, Any] = {}  # persistent across calls

def _compile_formula(formula: str):
    if formula in _compiled_cache:
        return _compiled_cache[formula]  # 直接复用，不重新编译

    ast_result = _PARSER.ast(formula)
    func = ast_result[1].compile()
    _compiled_cache[formula] = func
    return func
```

**关键设计决策：** 缓存在整个重算会话中持久化，不在每次 `recalculate()` 调用时清空。只有加载新 Excel 文件时才清空。

### 4.3 输入解析

```python
def _resolve_input_key(raw_key: str, formula_sheet: str, graph: FinancialGraph):
    # 处理跨sheet引用、表间别名、范围引用

    if "!" in raw_key:
        sheet_part, addr_part = raw_key.split("!", 1)
        sheet_part = _normalize_sheet_name(sheet_part, graph)
    else:
        sheet_part = formula_sheet
        addr_part = raw_key

    if ":" in addr_part:  # 范围引用
        return _resolve_range(sheet_part, addr_part, graph)

    cell_id = _addr_to_cell_id(sheet_part, addr_part)
    cell = graph.cells.get(cell_id)
    return np.array([[_coerce(cell.value)]])  # 转成 numpy 数组
```

### 4.4 支持的函数

`formulas` 库支持 **645 个 Excel 函数**，包括：
- 数学：SUM, SUMIF, AVERAGE, MAX, MIN, COUNT, ROUND
- 查找：VLOOKUP, HLOOKUP, INDEX, MATCH, OFFSET
- 逻辑：IF, IFERROR, IFS, AND, OR, NOT
- 财务：IRR, NPV, PV, FV, PMT, RATE, MIRR
- 日期：YEAR, MONTH, DAY, DATE, DATEDIF
- 文本：LEFT, RIGHT, MID, TEXT, CONCATENATE

## 五、与 Excel 计算结果的对比

### 5.1 一致性保障机制

| 机制 | 说明 |
|------|------|
| 公式库 | 使用 `formulas` 1.3.4 库，与 Excel 引擎对齐 |

- 某些古早 Excel 函数（如 XIRR 在部分边界情况）
- 条件格式、数据验证（不属于公式计算）
- 数组公式（部分支持）

## 六、性能优化

### 6.1 优化前后对比

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 20K 格重算时间 | ~11 小时 | ~7-15 分钟 |
| 主要原因 | 每次清缓存 + 重复编译 | 缓存复用 + dirty-tracking |
| 迭代轮数 | 固定 10 轮 | 直到收敛（通常 3-5 轮） |

### 6.2 关键优化点

1. **公式缓存持久化**：`clear_formula_cache()` 不再每次调用
2. **dirty-tracking**：只算值真的变了的后继格
3. **单次拓扑排序**：环外格只算一遍
4. **提前退出**：收敛后立即停止，不做无用迭代

## 八、测试验证

### 8.1 验证方法

1. 在 03_重算 页面修改参数（如上网电价±5%）
2. 对比重算结果与 Excel 原文件的计算结果
3. 检查 `result.error_cells` 是否为空
4. 检查 `result.scc_iterations` 是否合理（通常 < 10）

### 8.2 边界情况

| 场景 | 预期行为 |
|------|----------|
| 修改常量格（无公式） | `changed_cells` 为空，直接返回 |
| 修改后无下游依赖 | `changed_cells` 只有被修改的格本身 |
| 触发大环收敛 | `scc_iterations` > 1，`changed_cells` 包含多轮迭代中值变化的格 |
| 公式评估失败 | 格子加入 `error_cells`，不影响其他格计算 |
