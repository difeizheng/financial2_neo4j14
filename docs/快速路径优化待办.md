## 待实施（按收益排序）

### P1: IF（6,036 剩余）

**实际格式障碍：**
1. `=IF(K10=表!I7, 100%, H10)` — 百分号 `100%` 不是数字
2. `=IF(K10=表!I7, 1, 0)` — 等号比较符 `=` 与公式开头 `=` 冲突
3. `=IF(A1>B1, C1, D1)` — cell ref vs cell ref 比较
4. `=IF(ISBLANK(A1), 0, B1)` — 嵌套函数条件

**实施思路：**
- 百分号：`=IF(A=B, 100%, 0)` → 将 `100%` 解析为 `1.0`
- 扩展 IF 正则支持 `REF op REF` 比较（当前只支持 `REF op literal`）

**预期收益：** +14%（6,036 / 42,942）

### P2: SUMIF（1,002）

**实际格式：** `=SUMIF(range, criteria, sum_range)`
- 需要解析三参数，对 range 做条件过滤求和
- 复杂度中高，但收益可观

**预期收益：** +2.3%

### P3: EDATE（450）

**实际格式：** `=EDATE(start_date, months)`
- Excel 日期序列号 + 月份偏移
- 纯 Python 可实现

**预期收益：** +1.0%

### P4: COUNTIF（192）

**实际格式：** `=COUNTIF(range, criteria)`
- 范围条件计数

**预期收益：** +0.4%

### P5: 嵌套函数处理

**模式：** `=ROUND(DATEDIF(I5,I7,"D")/365*12, 0)`
- 需要先实现 DATEDIF 快速路径，再组合 ROUND
- 开发量大，收益不确定
