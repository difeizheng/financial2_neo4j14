# Excel财务模型诊断方法论

> 从两个真实调试案例中总结的分析方法论

## 问题1：如何在勾稽关系的Excel中定位差异根因

### 背景

修改时间序列表的静态日期为公式后，打开v3文件与源文件对比，发现折旧摊销表D2从966945.68变成0，以及大量关联cell值不一致。D2的公式是`=参数输入表!I217`，与时间表无直接关系，说明是全局性问题。

### 诊断过程

**阶段一：从症状回溯，缩小范围**

用户报告"表2-折旧摊销表D2从966945.68变成0"。D2的公式是`=参数输入表!I217`，跟时间表毫无关系。这说明不是局部问题，而是全局性问题。

**阶段二：逐层对比，隔离变量**

做了三层对比：
1. **公式文本对比** — v3和源文件的公式字符串完全一致，排除"公式写错"
2. **缓存值对比** — 用`data_only=True`加载v3，发现公式cell全部返回`None`
3. **公式链模拟** — 用Python模拟MIN公式链，逐列对比原始静态值，发现公式逻辑其实是正确的

这三步排除了"公式逻辑错误"的假设，把矛头指向了缓存值。

**阶段三：对照实验，确认根因**

关键一步是roundtrip测试——加载原文件、不做任何修改、直接save。结果缓存值照样丢失。这证明问题出在工具（openpyxl），不是我们的修改。

### 根因

openpyxl的设计限制：save时只写公式(`<f>`)不写缓存值(`<v>`)。即使不修改任何cell，保存后所有公式单元格的缓存值都会丢失。

### 解决方案

改用XML级别操作（zipfile + lxml），只修改目标sheet的目标cell，其他文件字节级复制。最终13个其他sheet的缓存值100%保留。

### 方法论

| 原则 | 方法 |
|------|------|
| 逐层对比 | 公式文本 → 缓存值 → 公式链模拟，逐层排除假设 |
| 对照实验 | roundtrip测试（不改任何东西）隔离工具缺陷 vs 代码缺陷 |
| 怀疑工具 | 当所有代码假设都被排除后，问题可能在工具本身 |

## 问题2：如何在陌生的财务模型中定位结构缺陷

### 背景

抽水蓄能财务模型，折旧年限从29年改为20年后，Python引擎重算结果与Excel重算结果差异42%（表2_32_D: 1079821 vs 756824.99）。

### 诊断过程

**第一步：比例线索**

差异比值 1079821/756824.99 ≈ 1.426，接近 29/20 = 1.45。这暗示差异跟折旧年限成正比——不是随机错误，而是参数传播不完整。

**第二步：依赖链追踪**

从变动的参数出发，沿公式链追踪：
```
I622(折旧年限) → D101(结束日期) → AG103(=D101) → SUMIF → 折旧金额
```

到AG103这一步，链路看起来没问题。但SUMIF的结果不对，说明SUMIF的输入（年份行/月数行）有问题。

**第三步：对比SUMIF的输入**

检查SUMIF依赖的两行数据：
- 年份行(row 102)：`=YEAR(日期行)` — 只有公式，值正确
- 月数行(row 104)：`=DATEDIF(前一日, 当日, "d")/30` — 只有公式，值正确
- 日期行(row 103)：**D-AF全是静态datetime值，不是公式**

这就是断裂点。当I622改变 → D101改变 → AG103跟着变，但D-AF的静态值纹丝不动。SUMIF按年汇总时，中间年份的日期没有跟着调整，月数计算就错了。

**第四步：模式识别**

发现16个SUMIF块都有相同结构：
- 首列(C)：`=C{start}` — 公式
- 中间(D到倒数第二列)：静态值 — **缺陷**
- 末列(AG)：`=D{start}` — 公式

这是典型的"只设了头尾公式、中间硬编码"的模式，建模者可能没考虑参数变化场景。

### 根因

时间序列表16个SUMIF块的日期行中，中间列是硬编码的静态日期值，而非公式。折旧年限变化时，首尾列的公式能响应变化，但中间列无法响应，导致SUMIF汇总结果错误。

### 解决方案

将所有静态日期替换为`MIN(DATE(YEAR(prev_col)+1, 12, 31), D$start)`公式，并扩展到C-AX覆盖完整运营期。使用XML级别操作保留其他sheet的缓存值。

### 方法论

| 原则 | 方法 |
|------|------|
| 比例分析 | 差异比值接近参数变化比例 → 传播链不完整 |
| 依赖链追踪 | 从变动参数沿公式链正向追踪，找到第一个"不响应"的cell |
| 公式 vs 值 | 一个cell是公式还是静态值，决定了它能否响应参数变化 |
| 模式识别 | 同一sheet内多个相似结构块，发现一个缺陷后检查所有同类块 |

## 通用诊断框架

```
异常现象
  ↓
比例分析：差异是否与参数变化成正比？
  ├─ 是 → 传播链不完整，沿公式链追踪断裂点
  └─ 否 → 全局性问题，检查工具/环境
       ↓
逐层对比：公式文本 → 缓存值 → 计算逻辑
  ├─ 公式不同 → 公式写错
  ├─ 公式同、缓存不同 → 工具问题（如openpyxl丢缓存）
  └─ 公式同、缓存同、值不同 → 计算顺序/环境差异
       ↓
对照实验：最小可复现测试
  ├─ roundtrip测试：排除工具缺陷
  └─ 逐步修改：定位引入问题的具体改动
```
