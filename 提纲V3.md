# 财务模型知识图谱 — 面向财务咨询公司 V3（商业故事版）

## 核心思路转变

| 维度 | 旧思路（产品视角） | 新思路（商业视角） |
|------|-------------------|-------------------|
| 开场 | "我们做了什么" | "你的业务可以不一样" |
| 痛点 | Excel 的技术缺陷 | 咨询业务的交付瓶颈 |
| 演示 | 功能操作 | 业务场景 + 结果 |
| 价值 | "节省时间" | "赢更多单、交更快、赚更多" |
| 结尾 | "合作方向" | "你的竞争壁垒" |

## 一、开场钩子：一个反直觉的问题（2 分钟）

> 目标：不是讲系统，而是讲一个让他们思考的问题

**话术建议**：

「问一个问题：你们团队做财务模型，最值钱的资产是什么？

不是 Excel 文件。不是模板。

是**团队脑子里的逻辑**——为什么这样建模、参数怎么取、依赖关系是什么。

但这些知识，目前都锁在文件和人的脑子里。项目一结束，就散了。

今天演示的东西，解决的就是这个问题——把财务模型的'隐形知识'变成'显性资产'。」

**过渡**：「用一个简单的例子说明。」

---

## 二、大图：从文件到资产（3 分钟）

```
[客户给的 Excel] ──解析──▶ [知识图谱] ──导回──▶ [可交付的 Excel]
                                  │
                          看得见 · 改得起 · 留得住
```

三句话：
1. **输入**：任何 .xlsx 文件，不需要模板化
2. **中间**：自动提取公式依赖、指标关系、逻辑链
3. **输出**：改完导回 Excel，还是一个能用的财务模型

**关键定位**：不是替代 Excel——咨询顾问的交付物还是 Excel。这是中间的**理解层和操作层**。

## 三、四个咨询业务场景（16 分钟）

> 核心变化：不讲"功能"，讲"场景"。每个场景 = 咨询顾问的真实工作流。

### 场景 1：新项目组接手——3 天理解变成 3 小时（4 分钟）

**业务场景**：新项目启动，拿到客户 50+ Sheet 的财务模型。初级顾问需要多久理解？

**演示**：
- 上传文件 → 自动解析 → 图谱展开
- 点击某个关键指标（如"动态总投资"），一键追溯上下游
- 展示：49 张表、2968 个指标、依赖关系一目了然

**业务价值**：
- 传统：高级顾问带人逐格讲，3-5 天
- 图谱：初级顾问自己看，3 小时
- **释放高级顾问产能，缩短 ramp-up 时间**

### 场景 2：方案汇报——让客户"看得见"你的专业（4 分钟）

**业务场景**：给客户做方案汇报，改了 20 个参数，客户问"为什么改这些"。传统方式是口头解释或列个清单。

**演示**：
- 打开快照对比页面，选"基准方案" vs "优化方案"
- 热力图展示变化分布
- 点击传播路径：改了 A → 影响 B → 传导到 C
- 导出对比报告

**业务价值**：
- 传统：口头解释，客户半信半疑
- 图谱：可视化传播链，客户看得懂
- **汇报更直观，客户信任度更高，更容易过会**

### 场景 3：参数调整——改了敢交（4 分钟）

**业务场景**：客户要求"假设融资成本降 50bp，对整体有什么影响"。改完后敢不敢直接交？不知道影响范围。

**演示**：
- 选择场景 → 修改一个参数 → 重算 → 结果面板
- 左编辑，右结果：修改了什么 → 影响了哪些 → 影响多大
- 支持多个方案并行对比

**业务价值**：
- 传统：改完手工抽查，心里没底
- 图谱：自动识别全部影响单元格，100% 覆盖
- **交付质量可验证，降低返工风险**

### 场景 4：客户问答——从翻表到秒答（4 分钟）

**业务场景**：客户在群里问"2026年折旧摊销总额是多少"。传统操作：打开 Excel → 找表 → 找行 → 找列 → 回复。

**演示**：
- 打开 QA 页面 → 输入问题 → 秒出结构化回答
- 展示：指标卡 + 折线图 + 数据表 + 数据来源
- 相关问题推荐

**业务价值**：
- 传统：2-5 分钟找数
- 图谱：秒级响应，带图表
- **客户感知专业度提升，响应速度是竞争力**

## 四、这不是工具，是你们的竞争壁垒（5 分钟）

> 核心变化：从"我们有什么"到"你能得到什么壁垒"

### 四个维度的壁垒

| 维度 | 传统咨询公司 | 用图谱的咨询公司 |
|------|-------------|-----------------|
| **交付速度** | 新人 3-5 天理解模型 | 3 小时上手 |
| **交付质量** | 改完手工抽查 | 自动全量验证 |
| **客户体验** | 口头/文字解释 | 可视化传播链 |
| **知识沉淀** | 文件+人脑 | 可复用知识图谱 |

### 长期价值：从"卖人头"到"卖资产"

```
传统模式：每个项目从零开始 → 成本随规模线性增长
图谱模式：知识持续积累 → 边际成本递减，壁垒递增
```

**话术建议**：

「咨询公司最大的资产不是人，是知识。但这个知识目前都是散的——在文件里、在人脑子里。

知识图谱的作用，是把这些散的知识**结构化、可复用、可传递**。

这意味着：你做的项目越多，图谱越厚，交付越快，成本越低。这是复利效应。」

## 五、技术可靠性（2 分钟）

> 快过，一页建立信任

- 公式解析：73% 覆盖，依赖方向 100% 正确
- 重算精度：#VALUE! 从 5557 降至 0
- 浮点精度：容差机制，避免误判
- 数据安全：本地部署，数据不外传
- 兼容性：任何 .xlsx 均可解析，不需要模板化

## 六、下一步：从小处开始（3 分钟）

> 目标：降低决策门槛，给出可执行的下一步

**三步走建议**：

1. **POC 验证**（2 周）
   - 拿一个你们正在做的项目文件
   - 免费解析，看效果
   - 你们自己判断有没有价值

2. **试点项目**（1-2 月）
   - 选 1-2 个项目的顾问试用
   - 对比传统方式的效率/质量差异
   - 收集反馈

3. **规模化部署**
   - 根据试点效果决定
   - 可按项目、按团队、或全公司推广

**话术建议**：

「不需要先做决定。拿一个你们现在的模型文件，我来解析，你们自己看效果。这是最直接的验证方式。」
